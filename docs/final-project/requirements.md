# Final Project Requirements

ผู้เรียนต้องส่งระบบ CMMS แบบ End-to-End ตามลำดับ: วิเคราะห์ requirement → user stories → roles → workflow → ER diagram → PostgreSQL → Supabase Auth/RLS/Storage → API test → Power Apps/Power Automate → Power BI → deployment → user/admin/backup manuals

## Deliverables

BRD, Functional/Non-functional Requirements, User Stories, Acceptance Criteria, Architecture/Data Flow/ER diagrams, Data Dictionary, API Specification, Security Design, RLS Matrix, Test Plan/Cases, UAT Checklist, Deployment/Rollback Guide, User Manual, Administrator Manual และ Troubleshooting Guide

## Acceptance Gate

สร้าง Ticket ได้, assignment และ status transition ถูกต้อง, รูป 3 ช่วงเข้าถึงได้ด้วย policy, requester/technician เห็นข้อมูลต่างกัน, dashboard แสดงยอดตรงกับ reporting view และ restore backup ผ่านการทดสอบ
