# Use Case Template

- **Actor / Preconditions:** ระบุผู้เกี่ยวข้องและข้อมูลที่ต้องมี
- **Trigger:** เหตุการณ์เริ่มต้น
- **Main Flow:** ขั้นตอนปกติที่ระบบและคนทำ
- **Alternative Flow:** ทางเลือกเมื่อข้อมูลยังไม่ครบหรือเลือก vendor
- **Exception Flow:** network failure, unauthorized, duplicate และ validation error
- **Business Rules:** priority, SLA และ status transition
- **Data Used:** ticket, asset, work order, photo และ audit log
- **Security:** authentication, authorization, RLS และ storage policy
- **Acceptance Criteria:** เงื่อนไขทดสอบที่สังเกตได้
- **KPI:** ตัวชี้วัดผลลัพธ์
