# Contributing

ขอบคุณสำหรับการช่วยปรับปรุงคู่มือ CMMS นี้

1. สร้าง branch ที่อธิบายงาน เช่น `docs/week-02-sql`
2. เขียนเนื้อหาเป็นภาษาไทยและใส่ศัพท์เทคนิคภาษาอังกฤษกำกับเมื่อจำเป็น
3. ตัวอย่าง SQL, Power Fx และ Mermaid ต้องตรวจสอบให้ copy ไปทดลองได้
4. รัน `npm run build` ก่อนเปิด Pull Request
5. ห้ามใส่ secret, service role key หรือข้อมูลส่วนบุคคลจริงใน repository
